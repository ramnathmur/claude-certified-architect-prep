01

What this is

A real course, not a cram sheet

Built on an AI‑first learning doctrine: understanding comes before exam tactics. Every lesson leads with the “why” before the “how,” names reusable patterns, anchors them to real production work, and tests with active recall instead of passive reading.

The project has two halves that work together. A self‑study library — faithful replicas of Anthropic Academy courses plus web‑researched, source‑cited “extension” lessons that take each topic to architect depth — all surfaced through a single dashboard. And a live teaching curriculum — eight guided modules you run with Claude as your professor, who orients you, teaches the mental model, sets a hands‑on exercise, then quizzes you and logs what you missed. Everything is self‑contained HTML that opens offline; nothing depends on a paid API key.

The five exam domains

Where the weight is — and where it's taught

CCA‑F is scored on a scaled basis: hard scenario and anti‑pattern items carry most of the points, so you study for judgment, not coverage. The exam draws from 6 architecture scenarios, 4 of which appear per sitting.

DomainWeightTaught in
1 · Agentic Architecture & Orchestration 27% Module 1 + Subagents course, Orchestration & Orchestrator‑Worker extensions
3 · Claude Code Configuration & Workflows 20% Module 2 + Claude Code Configuration extension
4 · Prompt Engineering & Structured Output 20% Module 3 + Prompt Engineering Depth extension
2 · Tool Design & MCP Integration 18% Module 4 + MCP and SDK‑primitive extensions
5 · Context Management & Reliability 15% Module 5 + Context Management & Caching extensions
05

Getting maximum value

The library is built. Now go to class.

The single biggest lever isn't more content — it's using the live teaching loop. Reading the HTML and clicking MCQs is the easy half; the deep learning lives in the parts most people skip. Here's the order of leverage.

⚠️

Read this honestly

The course library is complete and QA‑passed — but the live modules haven’t been run for real yet (the recorded baselines are placeholder/synthetic). That means your largest gains are still on the table. Everything below converts a beautifully built school into an education you actually received.

Do first

Hold class — run the live modules

Open Claude on the project and say Run Module 1 (Agentic Architecture, 27%), then Run Module 3 (Prompt Engineering, 20%) — 47% of the exam. Let the professor teach; let the quiz measure you, not a modelled answer.

Do first

Do the Build acts, not just the quiz

Design the multi‑agent system on paper, write the real CLAUDE.md, design the MCP server and its error surface. These hands‑on exercises are where exam knowledge becomes architecture judgment you can use at work.

Build the habit

Teach it back before you’re tested

Before each module quiz, explain the mental model in your own words. If you can teach it, you know it — if you can’t, you’ve found the gap before the exam does.

Build the habit

Pre‑read, then run the module on top

Skim the matching extension lesson from the Launchpad first, then run the live module — so the session deepens what you read instead of starting cold.

Keep it durable

Cycle your gaps with spaced review

Re‑surface open GAPS.md entries on a cadence; promote one to “Resolved” only after 3 clean recalls. This is the difference between knowing it today and knowing it on exam day.

Keep it durable

Anchor every domain to your real work

For each module, note where you’ll use it in your production Claude systems. Education that’s tied to real projects outlives the exam.
